# 练对啦 UI 风格基线

这份文件用于固定当前确认下来的基础 UI 方向，后续继续设计时默认遵守，除非用户明确要求改风格。

## 总体方向

- 目标风格：真实微信小程序个人中心风格，不做后台感，不做重装饰，不做 AI 模板味。
- 参考气质：接近用户提供参考图的轻量生活服务类小程序排版。
- 核心感受：轻、透气、规整、自然，不拥挤。

## 页面结构原则

- 先顶部个人信息，再功能入口，再内容模块。
- 模块顺序要自然，避免信息一上来就堆满整屏。
- 首屏优先放：
  - 顶部个人名片
  - 4 个核心快捷入口
  - 主要功能折叠卡片
- 二级信息默认折叠，一次只展开一个模块。

## 视觉风格

- 背景：浅色系，以白底和浅灰底为主。
- 卡片：白色卡片，轻阴影，圆角柔和。
- 强调色：暖橙色，少量用于激活态、数字、按钮和重点标签。
- 文本：
  - 标题用深色
  - 描述用中灰
  - 弱化说明用浅灰
- 避免大面积深灰黑底、厚重面板、渐变堆叠。

## 布局密度

- 页面必须保留明显留白。
- 模块之间靠留白区分，不靠深色堆块硬分层。
- 文字不要塞满一行，描述允许换行。
- 同屏元素数量要克制，避免“每块都想展示很多”。

## 组件约束

- 顶部：头像 + 昵称 + 身份标签 + 简短数据。
- 快捷入口：4 个一行，均匀排布，轻卡片。
- 主功能区：使用折叠卡片。
- Tab 栏：简洁、轻量、微信小程序感，不做后台导航感。
- 标签：小巧、淡底色、轻提醒，不做夸张徽章。

## 不再采用的方向

- 不使用“后台控制台式”侧边导航。
- 不使用深灰重卡片堆满整屏的布局。
- 不使用过多金色、蓝色混合强调。
- 不使用视觉噪音很高的复杂装饰。

## 适用范围

后续以下页面默认沿用这个方向：

- 我的
- 练习
- 练对
- 论坛
- 相关二级详情页

## 当前结论

当前基础 UI 风格已确认：

- 顺序可以继续微调
- 但风格方向先固定为：轻量、白底、微信小程序感、低拥挤度、参考图同类气质
