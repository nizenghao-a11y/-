# Phase 01 - 练对 Core

## Page Goal

练对是平台第一阶段的核心付费闭环：用户上传动作视频，平台托管资金，合格点评人接单交付，用户验收、公开案例或进入售后仲裁。

## User-Side Page Structure

练对首页只保留三层，避免入口杂乱：

1. 顶部公告提示栏：说明动作点评不是医疗诊断，有疼痛、麻木、术后恢复等情况会提高点评人门槛。
2. 主入口：`我要找人看动作`，大卡片承接大多数发单用户。
3. 细分场景入口：器械 / 基础动作 / 体态 / 篮球 / 康复，小卡片只负责快速进入发单，不承载复杂规则。
4. 个人订单入口：`我的进行中订单`，独立管理发单后的订单状态。

页面内容不放跨底部 Tab 的导流按钮。练习、论坛、我的由底部 Tab 承担。

## Publish Flow

发单流程固定为 8 步，可回退修改：

1. 选择点评类型：器械使用求教、基础动作评估、体态力线分析、肌肉形态 & 强弱分析、篮球专项动作点评、职业病康复、徒手动作纠错、动作代偿专项分析、呼吸模式 & 发力联动点评。
2. 选择动作 / 部位：肌群、专项和自定义动作名。
3. 上传视频：8-30 秒，选择正面 / 侧面 / 45 度 / 背面视角；勾选是否存在身体疼痛。
4. 描述问题：预设问题 + 自定义输入。
5. 公开方式：公开单 / 隐私单。
6. 选择点评人范围：根据订单类型、隐私、疼痛状态联动。
7. 确认价格和交付标准：9.9 / 19.9 / 39.9 三档。
8. 发布并资金托管：微信支付托管，订单进入待接单池。

## Reviewer Range Rules

- 小白用户禁止接单。
- 普通训练爱好者只能发单，认证训练爱好者可接低价小额普通动作单。
- 篮球训练师是独立专项身份。
- 隐私单不可选择认证爱好者范围，只能选择专业教练 / 专项训练师及以上。
- 职业病康复 / 疼痛类订单默认锁定康复防护类优先。
- 肌肉形态 & 强弱分析订单只开放给专业教练 / 专项训练师及以上。
- 呼吸模式 & 发力联动点评属于专业选项，只开放给专业教练 / 专项训练师及以上。
- 动作代偿专项分析按普通动作单处理，但可在后台提高审核关注度。
- 发单用户默认不进入接单广场；接单入口由后端身份和审核状态控制，不在用户端页面展示规则卡片。

## Delivery Standards

三个价格档位先保持低门槛，降低冷启动阻力：

- 认证爱好者 / 专业身份可接单：9.9，50-100 字。
- 专业教练 / 专项训练师可接单：19.9，120-250 字。
- 康复防护类优先：39.9，200-400 字。

每单必须包含结论：正确 / 基本正确 / 有问题 / 无法判断。

动作正确不是空订单。点评人仍需说明做对的点、可以保持的点和必要的下一步建议。

## Order Center

订单状态固定 6 个：

1. 待接单
2. 点评中
3. 待补充素材
4. 待验收
5. 已完成
6. 售后 / 仲裁

订单详情必须包含：视频、用户原始问题、点评人身份与等级、完整点评内容、创建时间、约定交付时效、验收按钮、申请退款、公开为案例开关、客服介入入口。

售后 / 仲裁状态下，高亮平台介入入口。

## Supplement Material Flow

补拍视频 / 补充素材是订单详情里的异常分支，不改动发单 8 步主流程。

触发链路：

待接单 -> 点评中 -> 待补充素材 -> 点评中 -> 待验收 -> 已完成 / 售后仲裁。

接单者端在点评中订单详情里可以点击 `要求补充视频/素材`，选择原因：

- 视频画面模糊，无法判断动作。
- 缺少正面视角，请补拍正面动作。
- 缺少侧面视角，请补拍侧面动作。
- 视频时长过短 / 动作不完整。
- 补充说明。

发单用户端在 `待补充素材` 状态下必须看到：

- 顶部状态：当前状态：待补充素材。
- 系统提示：点评人反馈的原因。
- 原视频和原始问题继续可查看。
- 主按钮：重新上传视频（补拍）。
- 提交按钮：提交补充素材。
- 次按钮：放弃本次订单（申请退款）。

规则：

- 同一订单最多补拍 2 次。
- 进入待补充素材后暂停计算交付时效，用户提交补充素材后恢复。
- 用户在待补充素材阶段可申请全额退款，无手续费。
- 补传视频沿用原订单公开 / 隐私设置，不二次选择。
- 职业病康复 / 疼痛类订单素材多次不全时，平台可主动介入提醒。

## Refund Rules

退款先用固定四档，后端和后台统一判断：

- 0%：交付完整。
- 30%：内容偏浅。
- 50%：缺关键项。
- 100%：敷衍、违规或未交付。

## Backend/Admin Dependencies

- 微信登录与用户身份。
- 视频上传、压缩、过期和隐私权限。
- 订单状态机。
- 点评人身份、资质审核、接单权限。
- 微信支付托管与结算。
- 退款仲裁证据记录。
- 公开案例脱敏。

## Acceptance Checks

- 首页只出现公告、主入口、场景入口、进行中订单，不堆复杂规则。
- 发单流程完整覆盖 8 步。
- 隐私单、职业病康复、肌肉形态 & 强弱分析、呼吸模式 & 发力联动点评会抬高点评人门槛。
- 订单页具备 6 状态、完整详情和补拍素材分支。
- 普通用户端不直接暴露接单广场和交付管理。
- 私信路径不存在。
- 动作正确也必须交付正确点说明。
